## 2. 安全特性亮点

### 2.1 本地加密：数据库文件即密文
- **卖点**：保险库文件 `secret_roy_vault.db.enc` 在磁盘上始终是 AES-GCM-256 密文，即使设备丢失或被取证，没有主密码也无法解开。
- **技术背书**：采用二进制信封格式（nonce + ciphertext + auth tag），PBKDF2 十万轮派生包装密钥，运行时解密到临时工作库，锁定即清理。

### 2.2 AEAD 同步：同步流量也加密
- **卖点**：同步到自托管服务器或局域网传输的每一条数据，都经过独立 AEAD 信封加密，服务端只存密文。
- **技术背书**：`sroy-sync:` 前缀信封，AES-256-GCM + HKDF 派生同步密钥，vault 级隔离防止跨库解密。

### 2.3 生物识别：快，但不妥协
- **卖点**：Face ID / 指纹 / Windows Hello 秒解锁，生物识别仅用于解锁本地密钥信封，主密码不以任何形式泄露给操作系统。
- **技术背书**：`local_auth` + `flutter_secure_storage` 双层保护，生物识别失败自动回退主密码输入。

### 2.4 敏感剪贴板：复制后自动清理
- **卖点**：一键复制密码或密钥后，系统剪贴板在定时周期后自动清空，防止恶意应用长期监听剪贴板。
- **技术背书**：SHA-256 哈希指纹比对机制，确保仅清理 SecretRoy 复制的特定内容，不影响用户其他复制操作。

## 3. 跨平台亮点

### 3.1 六平台全覆盖
- **卖点**：Android / iOS / Windows / macOS / Linux / Web 六端统一体验，一个账号体系通吃全部设备。
- **技术背书**：Flutter 单一代码库，共享业务逻辑与 UI 组件；仅在最底层（数据库 FFI、相机、通知）做平台适配，核心加密与同步逻辑 100% 跨平台复用。

### 3.2 LAN 同步：没有公网也能多设备
- **卖点**：家里或办公室同一 WiFi 下，两台设备输入 8 位配对码即可建立加密通道同步，完全不需要互联网。
- **技术背书**：UDP 广播自动发现同网段主机，HTTP claim 协商会话，X25519 椭圆曲线密钥交换生成一次性传输密钥。

### 3.3 离线优先：没网也能用
- **卖点**：所有数据默认保存在本地，不注册、不联网、不依赖任何云服务即可完整使用全部功能。
- **技术背书**：SQLite 本地持久化 + 条件同步架构；同步是「可选插件」，而非「必需依赖」。

### 3.4 桌面端效率优化
- **卖点**：大屏设备自动展开为左侧导航栏 + 右侧内容区，支持 `Ctrl+F` 全局搜索、`Esc` 快速关闭，效率媲美原生桌面应用。
- **技术背书**：`AppLayout` 三档断点（compact < 720px < medium < 1080px < expanded），`AppNavRail` 桌面布局 + 键盘快捷键事件监听。

## 4. 同步技术亮点

### 4.1 字段级 CRDT：冲突自动消解
- **卖点**：两个人同时编辑同一条账号记录的不同字段，数据会自动合并；即使编辑同一字段，也有确定性策略决定胜负，不会弹出让人困惑的「选 A 还是选 B」。
- **技术背书**：自研字段级 CRDT 合并引擎，`CRDTMergeEngine` + `TotpCredentialMergeEngine` 双引擎覆盖账号与 2FA 凭证；合并结果通过 `crdt_merge_invariants_test.dart` 不变量校验。

### 4.2 Hybrid Logical Clock：无中心时间戳
- **卖点**：无需依赖 NTP 或服务器时间，设备间通过 HLC 自动校准事件顺序，离线编辑后同步仍能正确排序。
- **技术背书**：HLC 混合逻辑时钟（物理时间 + 逻辑计数器），`test/models/hlc_test.dart` 验证时钟递增、冲突处理与 JSON 往返一致性。

### 4.3 字段级 LWW：细粒度到每个输入框
- **卖点**：传统密码管理器的「Last-Write-Wins」通常覆盖整条记录，SecretRoy 的胜负判定精确到单个字段，最大程度保留用户编辑。
- **技术背书**：每个字段携带独立 HLC 时间戳，合并时逐字段比较时钟向量，非冲突字段全部保留。

### 4.4 同步状态机：一眼看清同步健康
- **卖点**：同步状态不是简单的「成功/失败」，而是 `offline → syncing → synced → error → conflictRecovery` 完整状态机，用户随时知道数据在哪里。
- **技术背书**：`SyncService` 状态机通过 `sync_state_machine_test.dart` + `multi_device_sync_test.dart` 验证全部状态迁移路径。

## 5. 用户体验亮点

### 5.1 模板系统：一条记录，千变万化
- **卖点**：不只是「网站账号」，还支持银行卡、身份证、WiFi、软件许可证等 10 组内置模板，自定义字段、排序、图标，让每种数据都有专属结构。
- **技术背书**：`AccountTemplate` + `FieldPreset` 双模型，模板切换时旧字段自动保留到历史区，避免数据丢失；`template_edit_view_test.dart` 验证编辑器全路径。

### 5.2 密码生成器：强度可视，一键填充
- **卖点**：在账号编辑页直接呼出密码生成器，长度滑块 + 字符类型开关实时生成，0–100 分强度评分直观判断安全性。
- **技术背书**：`EnhancedCryptoService` 密码学安全随机源 + 熵值评估算法，`password_generator_sheet_test.dart` + `enhanced_crypto_service_test.dart` 双重验证。

### 5.3 Vault 健康检查：保险库定期体检
- **卖点**：自动扫描弱密码、重复密码、过期记录、缺失 2FA，给出综合评分和一键修复入口，让安全意识从「靠记忆」变成「看报告」。
- **技术背书**：`VaultHealthCalculator` 多维度评分引擎，`vault_health_calculator_test.dart` 覆盖全健康/全风险边界场景。

### 5.4 通知中心：所有动态一处汇总
- **卖点**：同步冲突、密码过期、弱密码提醒、体检报告全部汇总到通知中心，未读红点角标提醒，不遗漏任何安全事件。
- **技术背书**：`NotificationProvider` + `AppNotification` 双模型驱动，分类聚合展示；本地通知调度在 Android/iOS/macOS 原生推送支持。

## 6. 平台差异化说明

### 6.1 移动端（Android / iOS）—— 随身安全中枢
- **独有优势**：相机扫码导入 TOTP（`TotpQrScannerView` + `MobileScanner`），一键复制 + 生物识别秒解锁。
- **场景**：超市扫码支付时的 OTP、地铁上快速查密码、Face ID 戴口罩时的指纹回退。
- **限制说明**：
  - 禁止 loopback 地址作为同步服务器（iOS 沙盒限制）。
  - 本地推送通知原生支持（Android/iOS 均完美）。

### 6.2 桌面端（Windows / macOS / Linux）—— 效率工作站
- **独有优势**：左侧 `NavRail` 常驻导航、`Ctrl+F` 全局搜索、`Esc` 关闭弹窗、大屏幕网格布局、剪贴板图片解码导入 TOTP。
- **场景**：办公室大屏批量管理账号、IT 管理员维护团队模板、开发者在多显示器间快速检索。
- **限制说明**：
  - Windows / Linux 不支持系统级本地推送通知（应用内通知中心仍完整可用）。
  - Linux 生物识别不可用；Windows 依赖 Windows Hello 硬件。
  - 桌面端数据库依赖系统 SQLite 动态库（Windows 10/11 自带 `winsqlite3.dll`）。

### 6.3 Web 端 —— 当前状态与展望
- **当前状态**：Web 端因缺乏 SQLite WASM/FFI 配置，**核心功能（创建/打开数据库）暂不可用**，不建议作为宣传卖点。
- **技术背景**：`path_provider` + `sqflite` 在 Web 下无原生支持；`flutter_secure_storage_web` 降级为 `localStorage`，安全级别下降。
- **市场建议**：宣传中可提及「基于 Flutter 的 Web 潜力」，但需标注「Web 适配开发中」或暂不放于官网下载入口。

## 7. 与竞品差异化

| 维度 | 传统云密码管理器（1Password/Bitwarden 等） | SecretRoy |
|------|------------------------------------------|-----------|
| **数据主权** | 数据托管于厂商服务器，用户信任厂商安全能力 | **本地优先**，数据默认只存设备；同步可选自托管 `roy_server`，数据主权完全归用户 |
| **同步架构** | 中心化服务器做冲突裁决，离线时功能受限 | **可选 LAN 直连 + 服务端中继双模式**，字段级 CRDT 自动合并，离线可用度 100% |
| **开源与可控** | 闭源或部分开源，用户无法审计同步协议 | **开源友好架构**，540+ 单元测试公开核心逻辑行为契约，CRDT/加密/同步协议可审计 |
| **自托管门槛** | 自托管版本通常复杂（Docker/K8s/数据库集群） | **Node.js 单服务 `roy_server`**，零数据库依赖，启动即服务，适合个人/小团队 |
| **2FA 导入** | 多数仅支持手动输入或单一扫码方式 | **移动端扫码 + 桌面端剪贴板图片解码双路径**，全场景覆盖 2FA 迁移 |
| **密码健康** | 通常仅检测弱密码和泄露 | **弱密码 + 重复密码 + 过期记录 + 缺失 2FA + 不完整记录** 五维评分，一键跳转修复 |
| **架构债务** | 大型代码库常见大量 TODO/FIXME | **全项目 0 TODO / 0 FIXME / 0 HACK**，工程纪律严格，长期维护成本低 |
