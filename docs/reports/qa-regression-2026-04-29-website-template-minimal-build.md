## 1. 变更范围

本轮将唯一内置模板的产品语义从通用敏感信息容器调整为网站凭据模板。

保留点：

- 内部 `templateId` 继续使用 `generic_info`，避免历史账户变成模板缺失。
- 内置模板仍然只有一个，默认入口保持轻量。
- 自定义模板能力不变。

变化点：

- 内置模板展示名改为“网站模板”。
- 字段从单一 `content` 改为 `website`、`username`、`password`、`notes`。
- `password` 字段默认保密、必填，并走密码字段类型。
- 文档、教程、用户手册和测试说明同步更新为网站模板口径。

### 3.2 新建账户

| 用例 | 步骤 | 预期 |
|---|---|---|
| 默认模板可用 | 点击新增账户，选择网站模板 | 表单出现网站、账号、密码、备注字段 |

| 保存成功 | 填写 GitHub 示例并保存 | 账户进入账户库，摘要展示网站或账号信息，密码不明文展示 |

### 3.3 编辑历史账户

## 4. 最小包体验收

构建策略：

- Android release 使用 `--split-per-abi`，避免生成单个 fat APK。
- 使用 `--obfuscate` 和 `--split-debug-info=build\app\symbols`。
- 安装时优先选择设备对应 ABI 的 APK，例如多数现代 Android 设备使用 `arm64-v8a`。

产物记录：

| 产物 | 路径 | 大小 | 说明 |
|---|---|---:|---|
| arm64-v8a APK | `build\app\outputs\flutter-apk\app-arm64-v8a-release.apk` | 20.35 MB | 推荐现代设备安装 |
| armeabi-v7a APK | `build\app\outputs\flutter-apk\app-armeabi-v7a-release.apk` | 18.03 MB | 旧 32 位设备 |
| x86_64 APK | `build\app\outputs\flutter-apk\app-x86_64-release.apk` | 21.68 MB | 模拟器或 x86_64 设备 |
| Debug symbols | `build\app\symbols` | 11.67 MB | 崩溃符号化用，不能作为安装包发布 |

## 6. 残余风险

- 由于内部 `templateId` 保持 `generic_info`，代码层名字和产品显示名存在历史兼容差异；这是为了避免历史账户模板缺失。
- 旧账户如果只存在 `content` 字段，进入新网站模板后会看到历史字段，需要用户手动整理到网站、账号、密码、备注字段。
- 本文档记录的是客户端回归；服务器同步协议本轮没有改动。
