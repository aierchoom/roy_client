# 账户与模板回归测试清单

生成日期：2026-04-19
适用范围：`roy_client`
目标：验证账户与模板改造后，关键业务链路未出现数据丢失、删除越权、交互崩溃和风险提示缺失。

## 1. 本轮重点回归范围

- 模板删除保护
- 模板字段新增、编辑、删除
- 账户编辑保存
- 历史字段保留、删除、恢复
- 模板缺失账户保护
- 账户列表风险提示

## 2. 风险分级

### P0

- 新增字段或编辑字段导致页面崩溃
- 保存账户时静默丢失历史字段
- 被账户引用的模板仍可删除
- 模板缺失账户仍可直接保存

### P1

- 历史字段删除后无法恢复
- 切换模板后旧字段未进入历史字段区
- 账户列表漏显示 `模板缺失` 或 `历史字段 N`

### P2

- 复制全部信息未包含历史字段
- 历史字段展示顺序、文案或只读态异常

## 3. 必测场景

### 3.1 模板编辑

1. 新建模板并新增字段，保存成功且无 crash
2. 编辑已有字段，保存成功且字段内容正确更新
3. 删除未被引用模板中的字段，允许删除
4. 删除已被引用模板中的字段，弹出风险确认提示

### 3.2 模板删除

1. 删除未被账户引用的模板
   预期：删除成功
2. 删除已被账户引用的模板
   预期：删除失败，并提示模板仍被使用

### 3.3 账户保存与历史字段

1. 模板删除字段后，打开旧账户
   预期：旧字段出现在历史字段区
2. 模板删除字段后，直接保存旧账户
   预期：旧字段仍被保留，不被静默删除
3. 在账户编辑页将历史字段标记删除
   预期：字段进入待删除区
4. 在保存前恢复待删除字段
   预期：字段回到历史字段区
5. 标记删除后保存账户
   预期：该历史字段从账户数据中真正移除

### 3.4 模板切换与缺失保护

1. 编辑账户时切换模板
   预期：弹出风险确认，旧字段不自动迁移
2. 切换模板后保存账户
   预期：旧字段进入历史字段区，不被静默清空
3. 打开模板缺失账户
   预期：允许查看，禁止保存，并显示保护提示

### 3.5 列表与展示

1. 存在历史字段的账户显示 `历史字段 N`
2. 模板缺失账户显示 `模板缺失`
3. 复制全部信息时包含历史字段

## 4. 建议测试顺序

1. 模板字段新增、编辑、删除
2. 模板删除保护
3. 账户历史字段保留与删除
4. 模板切换和模板缺失保护
5. 账户列表风险提示
